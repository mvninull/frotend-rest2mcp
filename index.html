Converta QUALQUER API em um Servidor MCP

Sem escrever código. Sem desenvolvimento adicional. Apenas configure a URL da especificação e pronto — sua API estará disponível como ferramentas MCP para LLMs em segundos.

Começar agora ⚡ Conversão em segundos ✨ Zero código

Por que este projeto?

LLMs modernos têm capacidades incríveis, mas não conseguem interagir diretamente com APIs REST existentes. O Protocolo MCP (Model Context Protocol) resolve isso, mas exige que cada API tenha um servidor MCP dedicado.

O MCP Server Universal elimina essa barreira: basta fornecer a URL da especificação OpenAPI/Swagger da sua API, e o servidor gera automaticamente as ferramentas MCP correspondentes.

Problema: LLMs não conseguem chamar endpoints REST diretamente
Solução: Conversão automática de specs OpenAPI/Swagger para ferramentas MCP

✨ Destaques Fundamentais

  • Zero código: Apenas configure a URL da spec
  • Múltiplas APIs: Configure várias APIs no mesmo cliente
  • Conversão automática: Swagger 2.0 → OpenAPI 3.0

Como funciona?

No transporte stdio (padrão):

  1. Configurar MCP_SPEC_URL no cliente MCP
  2. Servidor baixa e converte a especificação da API
  3. Ferramentas MCP são geradas automaticamente
  4. LLM usa as ferramentas para chamar a API
  5. Servidor morre quando o cliente desconecta

Nota: Para transporte SSE/HTTP, o servidor pode rodar de forma independente (ex: em host remoto), sem depender do ciclo de vida do cliente.

Exemplo: Duas APIs, Um Servidor

Configure múltiplas APIs no mesmo cliente MCP. Veja a diferença entre uma API externa (Swagger 2.0 desatualizado) e uma API local (OpenAPI 3.0 atualizada).

🌐 PetStore API (Externa)

Spec: Swagger 2.0 (desatualizado)

MCP_SPEC_URL=https://petstore.swagger.io/v2/swagger.json
MCP_SERVER_NAME=PetStore API (Swagger 2.0)
⚠️ Swagger 2.0 desatualizado
O servidor converte automaticamente para OpenAPI 3.0 usando swagger2openapi

🏠 Loja API (Local)

Spec: OpenAPI 3.0 (atualizado)

MCP_SPEC_URL=http://localhost:8000/openapi.json
MCP_SERVER_NAME=Loja API (OpenAPI 3.0)
✅ OpenAPI 3.0 nativo
Usado diretamente sem conversão, mais rápido e confiável

💡 Configuração no VS Code (Múltiplas APIs)

{
  "mcp.servers": {
    "petstore": {
      "command": ".../venv/Scripts/python.exe",
      "args": ["main.py"],
      "env": {
        "MCP_SPEC_URL": "https://petstore.swagger.io/v2/swagger.json",
        "MCP_SERVER_NAME": "PetStore API"
      }
    },
    "loja-local": {
      "command": ".../venv/Scripts/python.exe",
      "args": ["main.py"],
      "env": {
        "MCP_SPEC_URL": "http://localhost:8000/openapi.json",
        "MCP_SERVER_NAME": "Loja API"
      }
    }
  }
}

Instalação

Escolha o método adequado para o seu sistema operativo:

# Instalação completa (Python + Node.js)
make install

# Apenas Python
make install-python

# Apenas Node.js
make install-node

# Limpar ficheiros temporários
make clean

# Iniciar Inspector (requer MCP_SPEC_URL)
make inspect
# Dar permissões de execução
chmod +x setup.sh

# Executar script de instalação
./setup.sh

O script cria o venv, instala dependências Python e Node.js globalmente.

# Executar no PowerShell
.\setup.ps1

O script cria o venv, instala dependências Python e Node.js globalmente.

Quick Start (Teste Local)

Use a API de teste incluída (loja_api.py) para experimentar o servidor MCP em minutos.

1 Iniciar API de Teste

# Terminal 1: Iniciar loja_api.py
./venv/Scripts/python loja_api.py

# API disponível em:
# Swagger UI: http://localhost:8000/docs
# OpenAPI JSON: http://localhost:8000/openapi.json

2 Configurar Variáveis de Ambiente

# Terminal 2 (PowerShell):
$env:MCP_SPEC_URL = "http://localhost:8000/openapi.json"
$env:MCP_SERVER_NAME = "Loja API"

3 Iniciar MCP Inspector

# Terminal 2:
./venv/Scripts/python main.py --inspect

# Inspector abrirá em http://localhost:6274

4 Testar Ferramentas

No Inspector, clique em "List Tools" para ver as ferramentas geradas automaticamente a partir da Loja API (listar produtos, criar, atualizar, deletar).
