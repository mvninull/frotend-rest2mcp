Converta QUALQUER API em um Servidor MCP

Sem escrever código. Apenas configure a URL da especificação e pronto — sua API estará disponível como ferramentas MCP para LLMs em segundos.

⚡ Conversão em segundos ✨ Zero código 🔗 Múltiplas APIs

🔗 Repositório no GitHub: https://github.com/mvninull/open-mcp

Instalação via Git

Clone o repositório e tenha tudo configurado em minutos.

1 Clonar o Repositório

git clone https://github.com/mvninull/open-mcp.git
cd open-mcp

2 Configurar Servidor MCP no Cliente

Após clonar, configure o servidor no ficheiro JSON do teu cliente MCP (VS Code, Claude, etc.):

{
  "mcp.servers": {
    "minha-api": {
      "command": "${workspaceFolder}/venv/Scripts/python.exe",
      "args": ["${workspaceFolder}/main.py"],
      "env": {
        "MCP_SPEC_URL": "https://petstore.swagger.io/v2/swagger.json",
        "MCP_SERVER_NAME": "PetStore API"
      }
    }
  }
}

💡 Dica: Podes configurar múltiplas APIs, cada uma com sua spec URL. O servidor será spawnado automaticamente pelo cliente.

3 Pronto! Verificar Instalação

O script instala automaticamente:

  • Python venv com fastmcp, httpx e pydantic
  • Node.js swagger2openapi globalmente

Configuração no Cliente MCP (JSON)

Adicionar a configuração no ficheiro JSON do teu cliente MCP preferido. O servidor será spawnado automaticamente quando o cliente precisar.

VS Code / OpenCode

Edite settings.json:

{
  "mcp.servers": {
    "minha-api": {
      "command": "${workspaceFolder}/venv/Scripts/python.exe",
      "args": ["${workspaceFolder}/main.py"],
      "env": {
        "MCP_SPEC_URL": "https://petstore.swagger.io/v2/swagger.json",
        "MCP_SERVER_NAME": "PetStore API"
      }
    }
  }
}

Claude Desktop

Edite claude_desktop_config.json:

{
  "mcpServers": {
    "minha-api": {
      "command": "/caminho/para/venv/Scripts/python.exe",
      "args": ["/caminho/para/main.py"],
      "env": {
        "MCP_SPEC_URL": "http://localhost:8000/openapi.json",
        "MCP_SERVER_NAME": "Loja API Local"
      }
    }
  }
}

💡 Múltiplas APIs? Sem problemas!

Basta adicionar mais entradas no JSON do cliente. Cada API é um servidor independente que aponta para uma especificação diferente.
